O que é JavaScript e como usá-lo

JavaScript é a linguagem que adiciona comportamento às páginas web: responde a ações do usuário, altera conteúdo dinamicamente e consome APIs. Em conjunto com HTML (estrutura) e CSS (estilo), JavaScript torna páginas interativas e dinâmicas.

Como incluir JavaScript

<!-- 1) Inline (rápido, educativo) -->
<button onclick="alert('Olá Mundo!')">Clique aqui</button>

<!-- 2) Interno (dentro da página) -->
<script>
function saudacao() { alert('Olá Mundo!'); }
</script>

<!-- 3) Externo (recomendado em produção) -->
<script src="script.js"></script>

Variáveis

Variáveis são recipientes nomeados que armazenam valores em memória — números, texto, booleanos, objetos, arrays e muito mais. Entender o escopo (onde a variável está disponível) e o tempo de vida (quando ela existe) é essencial para escrever código previsível e sem bugs.

Declarações comuns:

  • const — declara uma referência que não será reatribuída. Use por padrão quando o identificador não precisa mudar.
  • let — declara uma variável cujo valor pode mudar. Use quando for necessário reatribuir.
  • var — sintaxe antiga; tem escopo por função e hoisting que pode confundir. Evite em código moderno.

Boas práticas:

  1. Prefira const por padrão. Só use let quando precisar reatribuir.
  2. Use nomes descritivos e em inglês ou no idioma do projeto (ex.: totalPrice, userName).
  3. Evite variáveis globais — encapsule em funções ou módulos para reduzir efeitos colaterais.

Exemplo rápido: const maxRetries = 3; torna a intenção clara: esse valor não deve mudar durante a execução.

Tipos de Dados

JavaScript possui tipagem dinâmica: uma variável pode conter valores de diferentes tipos ao longo do tempo. Isso traz flexibilidade, mas também exige atenção para evitar comportamentos inesperados causados por coerção de tipos.

Tipos primitivos:

  • number — inteiros e floats (ex.: 42, 3.14).
  • string — texto delimitado por aspas (ex.: 'Olá').
  • boolean — true ou false.
  • null — valor intencionalmente vazio.
  • undefined — variável declarada sem valor.
  • symbol — identificadores únicos (usados raramente).

Objetos e arrays: objetos ({}) e arrays ([]) são estruturas de referência que podem armazenar múltiplos valores. Eles têm comportamento de referência — ao atribuir um objeto a outra variável, ambos apontam para a mesma referência.

Dicas práticas:

  1. Use typeof para inspeção rápida, mas lembre-se que typeof null === 'object' (peculiaridade histórica).
  2. Quando precisar de valores imutáveis, use cópias defensivas (Array.slice(), spread syntax) para evitar mutações não intencionais.
  3. Para validação mais robusta, considere bibliotecas como zod ou yup em projetos maiores.

Verificar Tipo de Dado:

Código do Demo
<input type="text" id="data-type-input" placeholder="Digite qualquer valor">
            <button onclick="checkDataType()">Verificar Tipo</button>
            <p id="data-type-output"></p>
function checkDataType() {
              const rawValue = document.getElementById('data-type-input').value;
              const value = rawValue.trim();
              const output = document.getElementById('data-type-output');
            
              if (value === '') {
                output.innerHTML = 'Entrada vazia é do tipo: string';
                return;
              }
            
              if (!Number.isNaN(Number(value))) {
                const numberValue = Number(value);
                output.innerHTML = `${value} é do tipo: ${typeof numberValue}`;
                return;
              }
            
              const lowered = value.toLowerCase();
              if (lowered === 'true' || lowered === 'false') {
                const bool = lowered === 'true';
                output.innerHTML = `${value} é do tipo: ${typeof bool}`;
                return;
              }
            
              output.innerHTML = `${value} é do tipo: ${typeof value}`;
            }

Operadores e Condicionais

Operadores são os blocos básicos para manipular e comparar valores. Saber quando usar cada tipo evita bugs e melhora a legibilidade.

Tipos de operadores:

  • Aritméticos: +, -, *, /, % (resto).
  • Comparação: === (igualdade estrita), !==, <, >. Use === sempre que possível para evitar coerção automática.
  • Lógicos: && (e), || (ou), ! (negador) — úteis para combinar condições.

Condicionais:

  • if / else — forma direta de executar código condicionalmente.
  • switch — útil para várias comparações de igualdade com um mesmo valor.
  • Operador ternário (condicao ? valorA : valorB) — bom para expressões simples, mas evite complexidade dentro de ternários.

Exemplo de boa prática: verifique tipos quando comparar valores que podem vir de fontes externas (inputs, APIs). Evite confiar em coerção implícita — transforme/normalize os dados antes de comparar.

Operadores:

// Aritméticos
let soma = 10 + 5; // 15
let subtracao = 10 - 5; // 5
let multiplicacao = 10 * 5; // 50
let divisao = 10 / 5; // 2
let resto = 10 % 3; // 1

// Comparação
let igual = 5 == "5"; // true (valor igual)
let estritamenteIgual = 5 === "5"; // false (valor e tipo iguais)
let diferente = 5 != "5"; // false
let maiorQue = 10 > 5; // true

// Lógicos
let e = (5 > 3) && (10 > 5); // true
let ou = (5 > 10) || (10 > 5); // true

Condicionais:

// if/else
let hora = 14;

if (hora < 12) { console.log("Bom dia!"); } else if (hora < 18) { console.log("Boa tarde!"); } else { console.log("Boa
  noite!"); }

// Operador ternário
let mensagem = (hora < 18) ? "Bom dia!" : "Boa noite!";

                output.innerHTML = 'Você é um idoso.';
              }
            }

Funções

Funções são unidades de comportamento: encapsulam código para que possamos reutilizá-lo, testá-lo e organizá-lo melhor. Dominar funções é essencial para programar de forma modular.

Formas comuns:

  • Declaração de função: function somar(a, b) { return a + b; }
  • Expressão de função: const fn = function() { }
  • Arrow function: const fn = (a) => a * 2 — sintaxe concisa e comportamento de this diferente.

Boas práticas:

  1. Mantenha funções curtas e focadas (uma responsabilidade por função).
  2. Evite efeitos colaterais (alterar variáveis externas) quando possível — prefira retornar valores.
  3. Documente os parâmetros e o valor retornado, especialmente em funções que serão reutilizadas.

Exemplo: const formatPrice = (value) => Number(value).toFixed(2); — clara intenção, entrada e saída previsíveis.

Calculadora Simples:

Código do Demo
<input type="number" id="num1" placeholder="Número 1">
            <input type="number" id="num2" placeholder="Número 2">
            <select id="operation">
              <option value="add">Somar</option>
              <option value="subtract">Subtrair</option>
              <option value="multiply">Multiplicar</option>
              <option value="divide">Dividir</option>
            </select>
            <button onclick="calculate()">Calcular</button>
            <p id="calculation-output"></p>
function calculate() {
              const num1 = Number.parseFloat(document.getElementById('num1').value);
              const num2 = Number.parseFloat(document.getElementById('num2').value);
              const operation = document.getElementById('operation').value;
              const output = document.getElementById('calculation-output');
            
              if (Number.isNaN(num1) || Number.isNaN(num2)) {
                output.innerHTML = 'Por favor, digite números válidos.';
                return;
              }
            
              let result;
              switch (operation) {
                case 'add':
                  result = num1 + num2;
                  output.innerHTML = `${num1} + ${num2} = ${result}`;
                  break;
                case 'subtract':
                  result = num1 - num2;
                  output.innerHTML = `${num1} - ${num2} = ${result}`;
                  break;
                case 'multiply':
                  result = num1 * num2;
                  output.innerHTML = `${num1} × ${num2} = ${result}`;
                  break;
                case 'divide':
                  if (num2 === 0) {
                    output.innerHTML = 'Erro: Divisão por zero!';
                    return;
                  }
                  result = num1 / num2;
                  output.innerHTML = `${num1} ÷ ${num2} = ${result}`;
                  break;
                default:
                  output.innerHTML = 'Operação desconhecida.';
              }
            }

Eventos

Eventos são como mensagens sobre ações que ocorrem na página: cliques, toques, envio de formulários, movimentos do mouse, teclas pressionadas, entre outros. Eles permitem que o JavaScript reaja e atualize a interface em tempo real.

Como funciona:

  1. Você registra um handler (função) para um tipo de evento em um elemento usando element.addEventListener('click', handler).
  2. Quando o evento acontece, o handler é chamado com um objeto event com dados úteis (p.ex.: event.target).
  3. Você pode cancelar comportamentos padrão com event.preventDefault() (útil em formulários) e interromper a propagação com event.stopPropagation() quando necessário.

Dicas didáticas:

  • Delegação de eventos: em vez de adicionar um listener a dezenas de elementos, adicione um ao elemento pai e verifique event.target. Isso melhora desempenho e simplifica gerenciamento dinâmico de elementos.
  • Evite lógica pesada dentro de handlers — desacople em funções pequenas e teste-as isoladamente.

<button id="click-btn">Clique Aqui</button>
            <input type="text" id="key-input" placeholder="Digite algo aqui">

            <p id="events-output"></p>

              const clickBtn = document.getElementById('click-btn');

              });

                mouseArea.style.backgroundColor = '#d4edda';
                output.innerHTML = 'Mouse sobre a área!';
              });
            
              mouseArea.addEventListener('mouseout', () => {
                mouseArea.style.backgroundColor = '#eee';
                output.innerHTML = 'Mouse saiu da área!';
              });
            });

DOM (Document Object Model)

O DOM é a representação em árvore do documento HTML que o navegador mantém em memória. Cada elemento HTML vira um nó que pode ser consultado e manipulado com JavaScript — isso permite criar interfaces dinâmicas.

Operações básicas:

  • Selecionar elementos: document.querySelector, getElementById.
  • Ler/alterar conteúdo: textContent (mais seguro) e innerHTML (permite HTML, cuidado com XSS).
  • Mudar atributos e classes: setAttribute, classList.add/remove.
  • Criar/remover nós: createElement, appendChild, remove.

Performance e segurança:

  1. Minimize reflows: agrupe atualizações ao DOM e faça manipulações em fragmentos (DocumentFragment) quando gerar muitos nós.
  2. Evite innerHTML com dados não confiáveis — sanitize entradas para prevenir XSS.

Exercício prático: crie um formulário que insere itens em uma lista; use preventDefault() para evitar o reload e appendChild para adicionar elementos dinamicamente.

Manipulação do DOM:

Código do Demo
<input type="text" id="dom-input" placeholder="Digite um texto">
            <button onclick="addElement()">Adicionar Elemento</button>
            <button onclick="changeColor()">Mudar Cor</button>
            <button onclick="clearElements()">Limpar</button>
            <div id="dom-output"></div>
function addElement() {
              const input = document.getElementById('dom-input');
              const output = document.getElementById('dom-output');
              const value = input.value.trim();
            
              if (value === '') {
                alert('Por favor, digite algum texto.');
                return;
              }
            
              const newElement = document.createElement('div');
              newElement.textContent = value;
              newElement.classList.add('dom-item');
            
              output.appendChild(newElement);
              input.value = '';
            }
            
            function changeColor() {
              const output = document.getElementById('dom-output');
              const elements = output.querySelectorAll('.dom-item');
              const colors = ['#e8f4fd', '#fff3cd', '#d1ecf1', '#d4edda', '#f8d7da'];
            
              elements.forEach((element, index) => {
                element.style.backgroundColor = colors[index % colors.length];
              });
            }
            
            function clearElements() {
              document.getElementById('dom-output').innerHTML = '';
            }

Interação com APIs e Fetch

O que é uma API e por que JSON é importante

APIs são contratos que permitem que sistemas conversem: um servidor expõe endpoints que retornam dados (por exemplo, usuários, posts, etc.). JSON é o formato padrão porque representa objetos de forma simples e é fácil de converter para estruturas JavaScript.

Operações comuns com APIs:

  • GET — busca recursos;
  • POST — envia dados para criar recursos;
  • PUT/PATCH — atualiza recursos existentes;
  • DELETE — remove recursos.

Boas práticas ao consumir APIs:

  1. Mostre um estado de carregamento durante requisições para melhorar a experiência do usuário.
  2. Trate códigos de erro HTTP (4xx/5xx) e exiba mensagens úteis.
  3. Use try/catch com async/await para lidar com erros de rede e parsing.
  4. Ao enviar dados em JSON, inclua o cabeçalho Content-Type: application/json e serialize com JSON.stringify.

  "usuarios": [

    { "id": 2, "nome": "Maria Santos", "email": "maria@exemplo.com" }

Usando fetch() na prática

Exemplo com async/await (mais legível):

async function loadData() {
            try {
              const res = await fetch('https://jsonplaceholder.typicode.com/posts');
              if (!res.ok) throw new Error('Erro: ' + res.status);
              const data = await res.json();
              console.log(data);
            } catch (err) {
              console.error(err);
            }
          }

Resumo: fetch() facilita requisições HTTP, mas você precisa tratar respostas, erros e estados de carregamento para construir interfaces robustas.

// Requisição GET simples
fetch('https://api.exemplo.com/usuarios')
  .then((response) => response.json())
  .then((data) => console.log(data))
  .catch((error) => console.error('Erro:', error));

// Requisição POST
fetch('https://api.exemplo.com/usuarios', {
  method: 'POST',
  headers: {
  'Content-Type': 'application/json'
  },
  body: JSON.stringify({
  nome: 'Novo Usuário',
  email: 'novo@exemplo.com'
  })
})
  .then((response) => response.json())
  .then((data) => console.log(data));

Buscar Dados da API:

Código do Demo

              <option value="posts">Posts</option>
              <option value="comments">Comentários</option>
              <option value="users">Usuários</option>

function fetchData() {
              const endpoint = document.getElementById('api-endpoint').value;
              const apiUrl = `https://jsonplaceholder.typicode.com/${endpoint}`;
              const resultDiv = document.getElementById('api-result');
            
              resultDiv.innerHTML = 'Carregando...';
            
              fetch(apiUrl)
                .then((response) => {
                  if (!response.ok) {
                    throw new Error('Erro na requisição: ' + response.status);
                  }
                  return response.json();
                })
                .then((data) => {
                  const limitedData = data.slice(0, 5);
                  resultDiv.innerHTML = `
${JSON.stringify(limitedData, null, 2)}
`;
              })
              .catch((error) => {
              resultDiv.innerHTML = `Erro: ${error.message}`;
              });
              }
            
              function clearApiResult() {
              document.getElementById('api-result').innerHTML = '';
              }
